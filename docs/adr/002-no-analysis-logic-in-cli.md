# ADR-002: CLI に分析ロジックを持たない

## ステータス

Accepted

## コンテキスト

MCP サーバーは 22 ツールでテクニカル指標計算、パターン検出、スコアリングを提供している。CLI でも同じ機能を提供するかの判断が必要になった。

## 決定

CLI は分析ロジックを一切持たない。bitbank API への薄いアクセス層に徹する。

## 理由

- **二重実装の回避。** MCP と CLI で同じ分析ロジックを維持するのは保守コストが高い
- **ターゲットユーザーの特性。** 中上級者は自分の指標・ロジックを使いたい人であり、固定の分析結果は不要
- **モデルの推論力を活用。** 生データを渡せば Opus・o3 等が任意の指標を計算できる
- **Agent Skills でカスタマイズ可能。** モデル側の分析ロジックを Skills として柔軟に定義できる

## 影響

- CLI 単体では「今買い時？」に答えられない（Skills またはモデルの推論が必要）
- MCP との役割分担が明確になる（MCP = 結論を渡す / CLI = 生データを渡す）
